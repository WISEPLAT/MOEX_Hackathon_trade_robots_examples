В директории представлены программные коды для приема и преобработки данных "Алгопак", а также последующего обучения нейросети для модели весов (используется в результирующем приложении).
Состав приложений:
1. модуль OneHotDay сбор данных.ipynb - сбор данных с Алгопак в цикле запросов для составления общего датасета с 1.01.2022 до 1.12.2023
2. модуль GoAlgo установка целевой переменной.lgp бесплатной общедоступной программы Логином (https://loginom.ru/download) для предобработки данных (включая нормализацию по максимуму-минимуму каждого тикета на периоде обучения) и получения результирующего датасета, готового к обучению нейросети.
3. модуль OneHotDay Обучение модели.ipynb - обучение нейросети на подготовленном датасете и получение файла весов, исчпользующегося в приложении "OneHotDay". 
4. модуль OneHotDay исполнение модели.ipynb - предварительное испытание алгоритма на входящих данных Алгопак (данные за 4 и 5 декабря 2023, не входящие в обучающий набор - т.е. чистая валидация модели. Получение метрик (по лучшей модели - 0.03137660537347649 по метрике MAPE на всех входящих данных прогноза и действительности на день торгов 5.12.2023). Подготовка задачи по реализации приложения на фронтенде.

Предполагается, что при действительном использовании приложения добавление данных, дообучение сети и получение действующей модели весов будет выполняться автоматически, в синхронизации с работающим приложением по завершению очередной торговой сессии и обновлению данных Алгопак.
Прилагаются также таблицы данных, включенных в модель - максимумы и минимумы по цене каждого тикета за весь период входных данных (для нормализацйии вновь получаемых данных при выполнении модели), а также данные по сектору экономики и краткого описания каждого тикета набора - файлы tickets_maxmin.csv и  tickers.csv соответственно.
